# Valid EARS Requirements

The system shall log all errors.

The app shall save user data to the database.

When a user logs in, the system shall display the dashboard.

When the timer expires, the app shall alert the user.

While the server runs, the app shall accept connections.

While the database is offline, the system shall queue incoming requests.

If the network drops, then the system shall retry the connection.

If a timeout occurs, then the app shall return an error code.

Where a camera exists, the app shall record video.

Where a microphone exists, the system shall capture audio.

## Non-Requirement Prose

This is a standard paragraph that describes the system architecture. The database stores records correctly. The interface allows users to view their data. It can perform all listed operations. This document serves as a reference.
